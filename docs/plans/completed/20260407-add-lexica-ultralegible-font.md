# Add Lexica Ultralegible Font

## Overview
Add Lexica Ultralegible as a fourth reader font option in CrossPoint Reader. Lexica Ultralegible is an accessible typeface (SIL OFL license) built on Atkinson Hyperlegible, designed for improved legibility — particularly for low-vision readers. It provides 4 styles (Regular, Bold, Italic, Bold-Italic) with 589 glyphs per style and 102 language support.

## Context (from brainstorm)
- **Font source**: OTF files from https://github.com/jacobxperez/lexica-ultralegible
- **License**: SIL Open Font License v1.1
- **Styles**: Regular, Bold, Italic, Bold-Italic
- **Sizes**: 12, 14, 16, 18pt (same as Bookerly/NotoSans)
- **Compression**: 2-bit greyscale + DEFLATE (same as other reader fonts)
- **Enum position**: `LEXICA = 3` (appended after `OPENDYSLEXIC = 2`)
- **i18n key**: `STR_LEXICA_ULTRALEGIBLE`
- **Web settings page**: auto-generated from API — no HTML changes needed

### Files/components involved
- `lib/EpdFont/scripts/convert-builtin-fonts.sh` — font conversion pipeline
- `lib/EpdFont/scripts/build-font-ids.sh` — font ID generation
- `lib/EpdFont/builtinFonts/all.h` — header includes
- `lib/EpdFont/builtinFonts/source/` — font source directory
- `src/main.cpp` — global font object registration
- `src/fontIds.h` — generated font IDs
- `src/CrossPointSettings.h` / `.cpp` — font family enum and ID mapping
- `src/SettingsList.h` — settings UI binding
- `lib/I18n/translations/*.yaml` — 21 translation files

### Patterns discovered
- Font conversion: loop over sizes × styles, invoke `fontconvert.py` with `--2bit --compress`
- Font objects: 4 `EpdFont` + 1 `EpdFontFamily` per size, inside `#ifndef OMIT_FONTS`
- Settings: `SettingInfo::Enum()` with `StrId` array auto-populates web UI dropdown
- Font IDs: SHA256-based hash generated by `build-font-ids.sh` using Ruby

## Development Approach
- **Testing approach**: Regular (code first) — this is primarily font integration, not logic
- No unit tests needed — font integration is verified by successful compilation and device testing
- Complete each task fully before moving to the next
- Make small, focused changes

## Solution Overview
Follow the exact same pattern used for Bookerly/NotoSans/OpenDyslexic:
1. Download OTF source files
2. Add conversion entries to build scripts
3. Run conversion to generate C headers
4. Register font objects in `main.cpp`
5. Wire up settings enum and font ID mapping
6. Add i18n strings to all translation files

No architectural changes needed — this is a straightforward addition following established patterns.
